JSON心得

发表于 2017-10-27 | 分类于 前端

JSON的总结

1: 是一种数据交换格式
2: 独立于编程语言
3：基于JS对象字面量表示法
4：表达数据的方式对通用的编程概念都很友好
注意：数据交换格式是一种在不同平台或者系统间传递数据的文本格式。而表示法是指一个用于诸如数字或单词等数据的字符系统。

一些值得思考的地方

字面量是什么意识

在编程语言中的literal(字面量)是一个名词，所谓字面量，就是对数据值的具体表示。
比如 x = 5; 这里的 x 是变量
x = x + 5; 这里我们肯定知道 x = 10了，其中的 5 就是字面量，也就是我们看到的实际值，就是字面的值。
比如5美元就是字面量，而行用卡就是变量

书写正确的格式

json中的名称一定要用双引号包裹，值不一定用，值可以是字符串、数字、布尔值、null、对象或者数组。
json中，多个名称-值对使用逗号分割
json媒体类型是application/json

验证json是否正确

使用JSONLint

从机器的角度去理解JSON

{ 指开始读取对象
} 指结束读取对象
[ 指开始读取数组
] 指结束读取数组
: 指在名称-值对中分割名称和值
，逗号指分割对象中的名称-值对，或者分割数组中的值，也可以认为是一个新部分的开始

所以如果书写不正确，就无法解析，比如你在名称-值对列表的结尾处加上一个逗号，就会报错，因为
你给机器的指令是”一个新部分的开始“，但是后面什么都没有，所以当然报错了

小知识

1: json不是JavaScript对象字面量，它只是基于JavaScript对象字面量，所以在JSON中，仅允许使用双引号来包裹字符串。
2：当值中含有双引号和反斜线等字符时，要用反斜线进行转义，不然会报错。
3：对象和数组很关键的一个区别就是，对象是名称-值对构成的列表或集合，数组是值构成的列表或集合。
4：数组中所有的值应具有相同的数据类型。
5：JSON中的null值的所有字母必须小写

JSON Schema

是对JSON数据格式的一致性进行验证，是数据发送方节约时间，保证数据正确的好工具
可以解决以下问题
1: 值的数据类型是否正确？ 可以具体规定一个值是数字、字符串等类型
2：是否包含所需要的数据？ 可以规定哪些数据是需要的，哪些是不需要的
3：值的形式是不是我需要的？ 可以指定范围，最小值和最大值

json安全问题

1：不要使用顶层数组，顶层数组是合法的JavaScript脚本，它们可以用

缘

世界上最神奇的就是巧合和缘分了，没有巧合，错过缘分，那也许在这个世界上，你将永远也见不到一些人和一些事物。

实习

正是因为存在神奇，很多事情才会以一种很美好的形式发生，既然很美好，那对你来说，必然可以使你愉悦，可以使你得到一丝成就与满足。实习来的很巧，静悄悄的，顺其自然的，在向我招手。

抉择

用产品经理的同理心去思考，我会选择将这份实习做好，将心比心，也许我一开始会很不适应，但是我会努力去适应，就是这么简单的思维，正是因为简单，所以不会变得错综复杂，生活不需要很复杂，只要努力了，是一定有收获的。

孤独的前行者

目测我在专业里的玩计算机编程的，好像几乎没有，即使有，也只是学的很浅。虽然前端开发相比后端开发，从算法能力和综合能力上有一些差距，但是我依然相信，一切的客观的事情都取决于人的行为。厉害的前端开发者很稀有，我希望我可以在前端行业独当一面，就够了。

感恩

在我学前端开发的时候，我学了一年多的产品经理，我非常感谢一个14届毕业的学长将我带上互联网这个行业，让我看到了互联网大致的全貌。一年多的产品经理的自学，我对整个互联网和IT行业有了很清晰的认识。在我学前端开发的时候，我遇到了很多贵人，很多公司的大牛给我很好的指引和建议。在这里我就不一一列出名字了，真的很感谢你们。

人生

很短暂，希望能在有限的时光里造出让自己满意的蔚蓝的天空，让自己变的更优秀。

爱情
