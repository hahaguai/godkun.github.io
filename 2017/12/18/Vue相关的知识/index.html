Vue相关的知识

发表于 2017-12-18 | 分类于 Vue | |

使用Vue过程中的点点滴滴的总结。

vue中带$前缀的变量名中的$是什么意识？
这些只是Vue的命名规则，为了区分普通变量属性，避免我们自己声明或者添加自定义属性导致覆盖，比如$component $http $set。
vue实例暴露了一些有用的实例属性和方法，它们都有前缀$，以便与用户定义的属性区分开来

router三要素
1：map关系，也就是映射关系 什么样的链接跳转到什么样的页面，什么样的区域显示什么样的页面。
2：link
3：页面中的链接

slot

子组件中的slot有name属性，与父组件的slot的值相对应，那么就会匹配到。没有匹配到的先会显示在默认的slot中。

子组件里有slot，父组件中的子组件里如果没有内容，则用slot里的内容代替，如果有，则显示父组件里的内容来替代slot

ref 和 refs

在父组件里的js中，调用子组件里的绑定事件要用 refs，即在子组件里设置
ref = “” 然后在父组件里的js中调用 this.refs.xxx.emit("")。
最经典的用法是：
父组件执行某个动作，该动作是调用子组件去触发一个自定义事件，然后触发这个事件后，在子组件里用this.$on('refresh', () => {})去监听这个事件，然后监听到就执行自己组件的一系列动作。

调用子组件里的方法要用this.refs.xxx.方法名()，因为组件就是一个对象，可以引用此组件里的方法。

可以用this.$refs.chart来获取DOM，这样可以减少DOM节点的消耗

router

组件内部页面跳转用this.$router.push({path: '/'})

每个vue实例在创建之前都要经过一系列的初始化过程，例如需要设置数据监听，编译，模板，挂载实例到DOM，在数据变化时更新DOM等。同时在这个过程中，也会运行一些叫做生命周期钩子的函数，给予用户机会在一些特定的场景下添加他们自己的代码。
注意：在mounted, updated, destroyed等，钩子的this指向调用它的vue实例

任何一个vue应用都是通过vue函数创建一个新的vue实例开始的，创建一个实例时，需要传入一个选项对象。当实例被创建时，它向vue的响应式系统中加入其data对象中能找到的所有的属性，并且当这些属性的值改变时，视图将会产生响应，即匹配更新为新的值。但是很重要的一点是，只有当实例被创建时，data中存在的属性才是响应式的，也就是说，这个时候你再添加一个新的属性，比如vm.b=’hi’，那么b的改动将不会触发任何视图的更新，所以如果你知道你会在晚些时候需要一个属性，那么很简单，就在一开始的设置它为空或者不存在，也就是设置一些初始值。

箭头函数

在vue实例中，不用箭头函数，因为箭头函数绑定父级上下文，也就是不绑定vue的实例对象。不要在选项属性或者回调上使用箭头函数，比如

都是不行的，因为this不是如你预期的vue实例 this.myMetho是未定义的

生命周期函数

钩子函数 对于前端来说，钩子函数就是指在所有函数执行前，我先执行的函数，即 钩住 我感兴趣的函数，只要它执行，我就先执行

watch 监听属性，只要属性变化，就立刻触发事件
watch方法只能监听一个data值的变化，而计算属性可以同时监听多个data值的变化，计算属性可以简化watch中重复的代码
当需要执行异步操作或者昂贵操作时，需要用到watch这个实例属性
watch里面可以有多个data值，但是每次只能监听一个

mounted 组件渲染完再执行，el被新创建的 vm.el替换，并挂载到实例上去之后调用该钩子

created 一个vue实例被生成后调用这个函数。因为一个vue实例被生成后还要绑定到某个html元素上，之后还要进行编译，然后再插入到document中。
每一个阶段都会有一个钩子函数，方便开发者在不同阶段处理不同逻辑。一般可以在created函数中调用ajax获取页面初始化所需

computed
计算属性的中的属性是关联当前组件的data里的属性的
一般将一些函数写成计算属性中的返回值，这样就在用emit触发时就不用在每个函数中触发
计算属性的依赖如果没有发生变化，当你再次调用计算属性的时候，能够立即返回上次缓存的计算值，而不需要从新执行计算属性的方法，而方法需要重新执行方法体

很短的知识点

  1. v-mode.lazy 彻底输入完才执行
  2. 引入组件时，需要注册
  3. 组件是一个大的对象
  4. v-for相当于 foreach(function(val, index){})
  5. 计算属性是可以缓存的，但是方法不能缓存
  6. transition组件中用name=”fade” 则自带fade-enter-in等
  7. 数据从哪创建的，就在哪修改，所以不能是父组件创建的，然后在子组件修改应该在子组件中用emit一个事件来让父组件修改
  8. 只要是插件，vue想使用的话，必须先用import加载进来，然后再用vue.use()进行注册,就相当于vue.prototype.$store = … 组件中可以直接用this.$store来获取
  9. 一般状态直接在子组件里加载，数据显示在父组件里加载
  10. 如果没有js文件里没有导入组件，在vue中却有这个组件，那就是全局组件，要在component目录下找对应的组件
  11. 我们在vue里面使用别人开发的组件，第一步就是install,第二步在main.js里面引入，第三步Vue.use这个组件
  12. 因为计算属性里的值不能传参，所以监听传参的数据必须用watch，如果不需要传参，应该在computed属性里进行监听
  13. 没有引用的话【也就是没有导入】就不会加载，就不会上线，对master代码库没有影响，就可以放在本地仓库里，不需要去删除它

自定义事件

自定义事件
onchange=”doSomethingOnSlideChange”

如果在子组件中的函数中用this.emit(‘onchange’, 参数)来触发事件，那么在父组件中事件绑定的函数是不能加括号的，也就是

onchange=”doSomethingOnSlideChange()” 这样写是错误的，以为这样写就是立即执行函数了，不是触发才执行的。

如果在子组件中传递了数据，那么父组件中的函数doSomethingOnSlideChange()中加参数的话就会关联子组件中的数据，如果不加参数就不会关联子组件中的数据

props

props 在父组件中定义两个变量，动态绑定到子组件中 子组件要显式地用 props 选项声明它期待获得的数据
var childNode = {
template: ‘

‘,
props:[‘message’]
}
message就是期待的数据

mixins

vue中mixins是为了解决属性或者其他值的复用的，比如dialog这种弹窗组件，因为里面一堆数据配置，如果你在其他组件用这个组件的话，就必须写很多属性和函数，很麻烦
用mixins就可以避免，直接导入。
mixins只能返回一个对象，但是可以在一个对象中定义多个对象，然后导出
extends

父组件和子组件进行通信的最好的原则

一般都是子组件CRUD后，触发自定义事件，自定义事件绑定了父组件里的方法，然后会自动触发父组件里对应的方法，由父组件进行CRUD的操作，然后再将操作后的值赋值给一个变量，比如envs，也就是更新了envs的数据。然后，在父组件vue文件里进行对子组件绑定属性，即:envs="envs"，然后通过最新的envs传递到子组件中的props{envs: {type: Array}}中，对子组件也进行更新展示。
以上操作的根本目的就是降低耦合，提高组件复用性，让子组件只是提高状态展示的功能，其他关于状态改变的操作全部放在父组件里完成，相当于子组件是木偶组件或者傻瓜组件，父组件是智能组件或者是容器组件。
