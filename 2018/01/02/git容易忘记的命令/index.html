git 回滚

git 回滚是指混滚到某个commit的阶段

  • 要先用 git log 查看commit 日志
  • 然后用 git --hard reset hash值
  • 就可以回滚到这个commit的版本了，注意是这个commit的版本，比如你的commit2 出现问题了，想回滚，那肯定是回滚到这个commit2的上个版本commit1

分支冲突

第一种处理方法
不能丢弃本地修改，因为其中的某些内容的确是我们需要的，此时需要对unmerged的文件进行手动修改，删掉其中冲突的部分

你在A分支上commit了，然后PR了，这时你的PR没有冲突的提示，可以合并。但是之后有人在你PR的目标上的相同位置做了修改，这个时候，你的PR就有冲突的提示了，不可以合并，需要先获取 远程最新的origin/master 分支的上的commitID，然后存在本地mstaer分支上的一个文件内，然后再用git merge origin/master 合并到本地的文件上。然后会出现冲突，再在冲突的文件内根据需求将一些内容删除掉，然后在git add git commit git push origin xxx 就不会再冲突了

第二种解决方法
如果我们确定远程的分支正好是我们需要的，而本地的分支上的修改比较陈旧或者不正确，那么可以直接丢弃本地分支内容，运行如下命令(看需要决定是否需要运行git fetch取得远程分支)：
